# EasyTab — спецификация Flutter-приложения
## (без кода)

---

## 1. Общая информация
**Название:** EasyTab  
**Цель:** Приложение для создания отчётов с поддержкой фото/видео и экспорта в Excel + HTML + ZIP.  
**Платформы:** Android (основная), iOS (опционально).

---

## 2. Стратегия хранения файлов (КРИТИЧЕСКОЕ ТРЕБОВАНИЕ)
Приложение **НЕ ИСПОЛЬЗУЕТ ОРИГИНАЛЫ** медиа-файлов из галереи. Алгоритм:
1. Пользователь выбирает фото/видео из галереи.
2. Приложение **СОЗДАЁТ КОПИЮ** в свою приватную папку (невидимую для других приложений и пользователя через стандартные файловые менеджеры).
3. Оригинал остаётся в галерее — нет риска случайного удаления.
4. Если одно и то же медиа нужно в двух проектах — создаются две отдельные копии (это нормально для устройства).
5. В файле проекта сохраняются **пути к копиям**, не к оригиналам.

---

## 3. Модели данных
### 3.1. MediaItem (медиа-файл)
Хранит информацию о фото/видео:
- `name`: Название файла (например, 001.jpg)
- `type`: MIME-тип (например, image/jpeg, video/mp4)
- `attention`: Флаг "внимание" (true/false)
- `originalName`: Оригинальное имя файла
- `localPath`: Путь к КОПИИ в приватной папке приложения

### 3.2. Answer (ответ на вопрос)
Хранит ответ и прикрепленные медиа:
- `text`: Текст ответа
- `attention`: Флаг "внимание" для всего ответа
- `media`: Список MediaItem (медиа-файлы)
- `isEmpty`: Флаг "пустой ответ"

### 3.3. Question (вопрос)
Хранит информацию о вопросе:
- `id`: Уникальный ID
- `text`: Текст вопроса
- `type`: Тип вопроса ("text" для текста, "media" для медиа)

### 3.4. Report (весь проект)
Единый объект для всего отчёта:
- `reportName`: Название отчёта
- `questions`: Список Question
- `answers`: Словарь (ключ — индекс вопроса, значение — список Answer)
- `mediaCounter`: Счётчик для именования медиа (например, {photos: 1, X: 5})
- `timestamp`: Время создания/изменения
- `folderPath`: Путь к папке проекта на устройстве

## 4. Структура папок на устройстве
Приватная папка приложения (получаем через `getApplicationDocumentsDirectory()`):
```
/reports/
└── {НазваниеОтчёта}/
    ├── report.json          ← ЕДИНЫЙ ФАЙЛ ПРОЕКТА
    ├── /photos/             ← Копии обычных медиа
    │   ├── 001.jpg
    │   ├── 002.png
    │   └── ...
    └── /X/                  ← Копии медиа с флагом "внимание"
        ├── 001.jpg
        └── ...
```

---

## 5. Формат файла проекта (report.json)
JSON-файл хранит всю информацию о проекте:
```
{
  "reportName": "Мой Отчёт",
  "questions": [
    {"id": 1, "text": "Название объекта", "type": "text"},
    {"id": 2, "text": "Фото объекта", "type": "media"}
  ],
  "answers": {
    "0": [{"text": "Дом 123", "attention": false, "media": [], "_empty": false}],
    "1": [{"text": "", "attention": true, "media": [{"name": "001.jpg", "type": "image/jpeg", "attention": true, "originalName": "photo_2026.jpg", "localPath": "путь/к/копии"}], "_empty": false}]
  },
  "mediaCounter": {"photos": 1, "X": 2},
  "timestamp": 1234567890
}
```

## 6. Экраны приложения и их функции
### 6.1. StartScreen (Главный экран)
- Градиентный фон (сиренево-фиолетовый)
- Заголовок "EasyTab" с подзаголовком "Excel Report Builder"
- Кнопки:
  1. **+ Create New Report** → перейти к выбору шаблона
  2. **Open Existing Report** → открыть существующий проект (выбрать папку с report.json)
  3. **Your Reports** → список отчётов (пока пустой)
  4. **Settings** → настройки
- Блок "Instructions" с краткой инструкцией по использованию

### 6.2. TemplateSelectScreen (Выбор шаблона)
- Поле ввода названия отчёта
- Блок выбора шаблона:
  1. **Built-in Template** — встроенный шаблон (5 вопросов: Название объекта, Дата осмотра, Статус оборудования, Комментарии, Фото/видео)
  2. **Upload your Excel template** — загрузить Excel-файл (.xlsx/.xls) с шаблоном:
     - Столбец A: Текст вопроса
     - Столбец B: Тип вопроса ("text" или "media")
     - Первая строка — заголовок, данные начинаются со второй
- Превью выбранного шаблона (первые 5 вопросов)
- Кнопка "Use Template" для создания проекта

### 6.3. FormFillScreen (Заполнение форм — список)
- AppBar с названием отчёта и кнопкой "+ Add Question"
- Список карточек вопросов:
  - Номер и текст вопроса (редактируемый)
  - Список ответов:
    - Текстовое поле для ответа
    - Флаг "!" (attention) — меняет цвет рамки и фона
    - Кнопка "×" для удаления ответа (если их больше одного)
    - Секция с медиа: миниатюры, кнопка добавления
  - Кнопка "+ Add Answer" для добавления нового ответа к вопросу
- Нижняя панель (BottomNavigationBar) с кнопками:
  1. **Back** — вернуться на главный экран
  2. **Save Draft** — сохранить проект в report.json
  3. **Export** — экспортировать отчёт:
     - Сохраняет HTML-отчёт
     - Сохраняет Excel-отчёт
     - Создаёт ZIP-архив со всеми файлами

### 6.4. FormCardScreen (Заполнение форм — карточки)
- Альтернативный вид: вопросов в виде сетки (2 колонки)
- Каждая карточка показывает номер, текст вопроса и первый ответ

### 6.5. ReportsScreen (Список отчётов)
- Пока пустой экран с предложением создать новый отчёт

### 6.6. SettingsScreen (Настройки)
- Настройки папок: Templates, Reports, Media
- Информация о платформе (Android/iOS)
- Блок "About" с версией приложения
- Кнопки "Save Settings" и "Reset to Default"

## 7. Зависимости (пакеты Flutter)
Список необходимых пакетов для работы:
1. `path_provider` — получить путь к приватным папкам приложения
2. `image_picker` — выбрать фото/видео из галереи
3. `excel` — читать/создавать Excel-файлы
4. `archive` — создавать ZIP-архивы
5. `provider` — управление состоянием приложения
6. `shared_preferences` — сохранять настройки
7. `file_picker` — выбирать файлы/папки

## 8. Ключевые сценарии использования
### 8.1. Создание нового отчёта
1. Открыть приложение → нажать "+ Create New Report"
2. Ввести название отчёта
3. Выбрать шаблон (встроенный или Excel)
4. Выбрать папку для сохранения
5. Заполнить ответы и добавить медиа
6. Сохранить черновик → экспортировать

### 8.2. Открытие существующего отчёта
1. Нажать "Open Existing Report"
2. Выбрать папку с проектом
3. Приложение загружает report.json и копии медиа
4. Пользователь продолжает редактировать

### 8.3. Экспорт отчёта
1. Нажать "Export"
2. Приложение сохраняет:
   - report.html — HTML-отчёт с таблицей и медиа
   - report.xlsx — Excel-отчёт с вопросами, ответами и путями к медиа
   - {НазваниеОтчёта}.zip — ZIP-архив со всеми файлами

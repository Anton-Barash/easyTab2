### Структура проекта

Проект организован по стандартной Flutter-архитектуре с разделением на слои. Вот детальная структура (на основе файловой системы и кода):

```
lib/
├── main.dart                          # Точка входа: инициализация провайдеров, локализации, маршруты
├── screens/                           # Экраны приложения (UI)
│   ├── template_select_screen.dart    # Экран выбора шаблона отчета (поля для типа изделия, фабрики, модели; загрузка заголовка)
│   ├── form_fill_screen.dart          # Экран заполнения формы (вопросы, медиа, разные режимы просмотра)
│   └── reports_screen.dart            # Экран списка отчетов (поиск, экспорт, управление)
├── providers/                         # Управление состоянием (Provider паттерн)
│   ├── report_provider.dart           # Основной провайдер: загрузка/сохранение отчетов, экспорт в Excel, сжатие медиа
│   ├── settings_provider.dart         # Настройки приложения
│   └── locale_provider.dart           # Управление языком (локализация)
├── models/                            # Модели данных
│   └── report_models.dart             # Классы: Report, Question, MediaItem и т.д. (сериализация в JSON)
└── l10n/                              # Локализация (генерируется автоматически из ARB-файлов)
    ├── app_localizations.dart         # Класс локализаций
    └── ...                            # ARB-файлы для языков

android/, ios/, web/, windows/         # Платформенные конфигурации
assets/                                # Статические ресурсы (изображения и т.д.)
build/                                 # Сгенерированные файлы сборки
pubspec.yaml                           # Зависимости и конфигурация
analysis_options.yaml                  # Настройки линтера
```

- **Общий стиль**: MVVM-подобный с Provider. Нет разделения на domain/business logic — все в провайдерах.
- **Размер**: ~3000+ строк кода, средний размер файлов (main.dart ~300 строк, screens ~500-1000 строк).
- **Зависимости**: 15+ пакетов, включая медиа (image_picker, video_player), файлы (file_picker), Excel (excel_community).

### Описание UI

Приложение имеет минималистичный, "ретро"-стиль UI с точечным фоном, серыми тонами и Material 3. Основной цвет — синий (#2563eb), фон — светло-серый (#f8f7f2). Поддержка темной темы отсутствует.

1. **StartScreen (главный экран)**:
   - Центрированный белый контейнер с закругленными углами и рамкой.
   - Верхний правый угол: переключатель языка (иконка глобуса, popup с EN/RU/ZH).
   - Три кнопки: "Создать новый отчет" (переход к выбору шаблона), "Продолжить отчет" (загрузка последнего), "Ваши отчеты" (список отчетов).
   - Нижняя часть: инструкции и версия.
   - Фон: точечный паттерн (CustomPainter).

2. **TemplateSelectScreen (выбор шаблона)**:
   - AppBar с заголовком.
   - Форма с полями: "Тип изделия", "Фабрика", "Модель" (TextField с рамками).
   - Загрузка изображения заголовка (кнопка, image_picker).
   - Список вопросов по умолчанию (локализованные: название объекта, дата, статус, комментарии).
   - Кнопка "Далее" для перехода к заполнению.

3. **FormFillScreen (заполнение формы)**:
   - AppBar с заголовком отчета.
   - Переключатель режима: список/карточки (ViewMode).
   - Список вопросов: текстовые поля, загрузка медиа (изображения/видео через picker), чекбоксы для внимания.
   - Боковая панель: превью медиа, навигация по вопросам.
   - Автосохранение с debounce (Timer).
   - Кнопки: экспорт, сохранение.

4. **ReportsScreen (отчеты)**:
   - AppBar с заголовком.
   - Поле поиска (TextField).
   - Список отчетов (ListView): дата, имя, действия (открыть, экспорт, удалить).
   - Экспорт: в Excel с архивом медиа (share_plus).

UI адаптивен для мобильных, но не оптимизирован для планшетов/десктопа. Навигация через named routes. Все тексты локализованы, но не все (см. ошибки выше).
